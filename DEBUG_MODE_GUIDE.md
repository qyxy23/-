# 海龟汤AI增强 - 调试模式使用指南

## 概述

为了便于开发和调试，海龟汤AI增强功能支持调试模式。在调试模式下，系统不会调用真实的AI服务，而是使用预设的模拟响应数据。

## 配置说明

### 启用调试模式

在 `application.yml` 文件中配置：

```yaml
haiqutang:
  ai:
    debug-mode: true  # 启用调试模式
```

### 禁用调试模式（使用真实AI）

```yaml
haiqutang:
  ai:
    debug-mode: false  # 使用真实AI服务
```

## 模拟数据文件

调试模式使用以下文件中的模拟数据：

**文件路径**: `src/main/resources/temp/aiResponse.txt`

**文件内容**: 包含完整的海龟汤游戏生成数据，包括：
- 进度任务列表
- 关键线索列表
- 主持人手册内容

## 工作原理

### 调试模式下的流程

1. **接口调用**: 用户调用 `/enhanceTurtleSoup` 接口
2. **Prompt生成**: 系统正常生成AI提示词
3. **数据源选择**: 检测到调试模式启用，跳过真实AI调用
4. **模拟响应**: 直接从 `aiResponse.txt` 文件读取预设响应
5. **数据处理**: 正常解析和处理模拟数据
6. **返回结果**: 返回结构化的增强结果

### 真实模式下的流程

1. **接口调用**: 用户调用 `/enhanceTurtleSoup` 接口
2. **Prompt生成**: 系统生成AI提示词
3. **AI调用**: 调用火山方舟AI服务
4. **响应处理**: 清理和解析AI响应
5. **返回结果**: 返回结构化的增强结果

## 日志输出

调试模式下会输出特定的日志信息：

```
INFO  - 使用模拟AI响应，调试模式已启用
INFO  - 成功读取模拟AI响应，文件大小: XXX 字节
```

真实模式下会输出：

```
INFO  - 调用真实AI服务
INFO  - AI响应长度: XXX
```

## 切换注意事项

1. **配置文件修改**: 修改 `application.yml` 后需要重启应用
2. **模拟数据更新**: 如需更新模拟响应，直接编辑 `aiResponse.txt` 文件
3. **数据格式**: 模拟数据必须符合JSON格式，确保能够被正确解析
4. **备用方案**: 如果文件读取失败，系统会使用简单的备用模拟响应

## 开发建议

## 故障排除

### 模拟文件不存在

**错误日志**: `模拟AI响应文件不存在: temp/aiResponse.txt`

**解决方案**:
1. 确认文件存在于正确路径
2. 检查文件名是否正确
3. 系统会自动使用备用模拟响应

### JSON解析失败

**错误日志**: `解析AI响应失败`

**解决方案**:
1. 检查 `aiResponse.txt` 文件的JSON格式
2. 确保没有未转义的特殊字符
3. 查看详细错误日志定位问题

### 配置不生效

**可能原因**:
1. 应用未重启
2. 配置文件格式错误
3. 配置路径错误

**解决方案**:
1. 重启应用
2. 检查YAML缩进和语法
3. 确认配置在正确位置
