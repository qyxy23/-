# 海龟汤智能游戏平台 - 项目介绍

## 项目概述

海龟汤智能游戏平台是一个基于向量化技术的智能海龟汤游戏系统，通过AI技术实现线索的智能拆解、向量化存储和智能匹配。当玩家提出问题时，系统会将问题向量化并与已有的向量化线索进行匹配，从而提供更准确的游戏体验。

## 核心技术架构

### 1. 向量化技术栈
- **向量化引擎**: BGE-small-zh-v1.5 模型 (512维向量，降维后为256维)
- **向量存储**: Redis Stack (高性能向量数据库)
- **向量匹配**: 余弦相似度算法
- **向量元数据管理**: MySQL + JPA

### 2. AI服务集成
- **AI模型**: 支持多种AI服务商 (OpenAI, 百度千帆等)
- **智能拆解**: AI自动分析汤底，生成结构化线索片段
- **智能问答**: 基于向量检索的上下文感知回答

## 核心业务流程

### 海龟汤创建流程
```
用户输入汤面、汤底、线索 → AI拆解汤底并生成线索片段 → 向量化所有内容 → 存储到Redis和MySQL
```

1. **汤面汤底向量化**: 将海龟汤的汤面、汤底、主持人手册分别向量化存储
2. **线索智能拆解**: AI分析用户提供的线索和汤底，生成不同推理层级的线索片段
3. **片段向量化**: 每个线索片段都被向量化并存储到Redis中，支持快速检索
4. **元数据管理**: 在MySQL中管理向量元数据，支持向量的生命周期管理

### 游戏问答流程
```
玩家提问 → 问题向量化 → Redis向量搜索 → 匹配相关线索片段 → AI生成回答 → 返回结果
```

1. **问题向量化**: 实时将玩家问题转换为256维向量
2. **智能检索**: 在指定海龟汤范围内搜索最相关的线索片段
3. **相似度计算**: 基于余弦相似度计算问题与线索的匹配度
4. **上下文构建**: 将最相关的线索片段作为上下文传递给AI
5. **智能回答**: AI基于检索到的上下文生成准确的回答

## 数据模型设计

### 核心实体

#### 1. HaiGuiSoup (海龟汤主体)
- 汤面、汤底、主持人手册
- 创建者信息、播放统计
- 线索拆解配置

#### 2. ClueFragment (线索片段)
- AI拆解生成的结构化线索
- 向量数据、推理层级、重要性评分
- 关联任务ID、触发关键词

#### 3. VectorMetadata (向量元数据)
- Redis键管理、向量类型
- 向量维度、创建时间
- 支持多种向量类型：SURFACE、BOTTOM、MANUAL、CLUE

### 向量类型分类
- **SURFACE**: 汤面向量，用于海龟汤推荐
- **BOTTOM**: 汤底向量，用于深度理解
- **MANUAL**: 主持人手册向量，用于游戏指导
- **CLUE**: 线索片段向量，用于问题匹配

## 技术实现亮点

### 1. 混合向量存储策略
```java
// Redis键结构设计
hai_gui:vec:surface:{soupId}     // 汤面向量
hai_gui:vec:bottom:{soupId}      // 汤底向量
hai_gui:vec:manual:{soupId}      // 主持人手册向量
hai_gui:vec:clue:{clueId}        // 线索片段向量
hai_gui:soup:{soupId}:fragment:{fragmentId}  // 海龟汤片段向量
```

### 2. 智能线索拆解
- **AI驱动的线索分析**: 自动分析线索类型、难度、重要性
- **多层级推理**: 支持表层、中层、深层三个推理层级
- **核心线索识别**: AI自动识别标记关键线索片段

### 3. 高性能向量搜索
```java
// 向量匹配示例
Map<String, Double> searchResults = redisClient.searchSimilarVectors(
    queryVector,    // 查询向量
    redisKeys,      // 搜索范围
    topK            // 返回数量
);
```

### 4. 上下文感知问答
- **多源上下文融合**: 结合汤面、汤底、相关线索片段
- **智能上下文排序**: 基于相似度评分的上下文优先级
- **回答类型识别**: AI自动判断回答类型（YES/NO/PARTIAL）

## API接口设计

### 核心接口

#### 1. 海龟汤管理
- `POST /api/haigui/soup/create` - 创建海龟汤（自动向量化）
- `PUT /api/haigui/soup/update` - 更新海龟汤（重新向量化）
- `DELETE /api/haigui/soup/delete/{soupId}` - 删除海龟汤（清理向量数据）

#### 2. 游戏问答
- `POST /api/game/question` - 处理玩家问题（向量检索+AI回答）
- `GET /api/game/clues/{soupId}` - 获取海龟汤线索
- `POST /api/game/match` - 智能线索匹配

#### 3. 向量搜索
- `GET /api/vector/search/soup` - 搜索相似海龟汤
- `GET /api/vector/search/clues` - 搜索相关线索
- `POST /api/vector/batch/vectorize` - 批量向量化

## 性能优化策略

### 1. 向量存储优化
- **Redis集群**: 支持向量数据的分布式存储

### 2. 搜索性能优化

### 3. AI服务优化

- **并发调用**: 支持多AI服务的并发调用
- **降级策略**: AI服务异常时的备选方案
